# nest.js-introduction

The **nest.js-introduction** project is designed as a foundational exercise to help its developer to
build and strengthen their understanding of the NestJS framework. This application covers key concepts
and best practices for building efficient, scalable, and maintainable applications using NestJS.

The primary goal of this project is to provide hands-on experience with various core features of NestJS, including:

- **Modular architecture**: Learn how to structure your application using NestJS modules to promote code organization and reusability.
- **Dependency Injection**: Understand how to manage application components using NestJS’s powerful dependency injection system.
- **Controllers and Services**: Get comfortable with the flow of request handling and how to implement services for business logic.
- **Middleware and Interceptors**: Explore how to manipulate request/response handling and integrate additional functionality into the application lifecycle.
- **Exception handling and Validation**: Learn best practices for error management and data validation using built-in mechanisms.

This project is intended for developers who are new to NestJS or looking to reinforce their skills by working through a practical, real-world example.

## Prerequisites

Before running the project, make sure you have the following installed:

- [Node.js](https://nodejs.org/) (latest version)
- [Typescript](https://www.typescriptlang.org/) (latest version)
- [NestJS](https://nestjs.com/) (latest version)

## Installation and Setup

In order to get this application up and running on your local machine, follow the
steps below.

1. Clone the repository from GitHub:

   ```shell
   git clone https://github.com/saeedNW/nest.js-introduction.git
   ```

2. Navigate to the project directory:

   ```shell
   cd nest.js-introduction
   ```

3. Install project dependencies:

   ```shell
   npm install
   ```

Note that the application default Listing port is `3000`.

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# build production
$ npm run build

# production mode
$ npm run start:prod
```
